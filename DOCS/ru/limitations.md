# Ограничения и security-заметки

## Ограничения

- **Медленные сканеры проходят.** Traps — rate-метры; источник, обращающийся медленнее
  порога (скажем, раз в минуту), его не превышает. Такие сканы — безвредный фоновый
  шум; для их обнаружения потребовалось бы per-packet-логирование, которого в прошивке нет.
  Исключение — abuse, распределённый по многим адресам *одного диапазона*: subnet trap
  ловит его, когда суммарный rate достаточно высок, а `COMPACT` сворачивает блок, который
  медленно наполняется одиночными банами.
- **Только TCP.** Traps срабатывают на TCP `SYN`. UDP-сканы/флуды Tacet не меряет — хотя, как
  показывают DoS-тесты в [validation](validation.md), UDP-флуд на роутер дропается дёшево
  базовым firewall и не создаёт заметной нагрузки.
- **Бан не разрывает активные сессии к роутеру.** `INPUT` drop исключает
  `ESTABLISHED`/`RELATED`, так что ваши tunnels сохраняются; разрываются только *новые*
  соединения от забаненного источника. Forwarded-потоки разрываются сразу.
- **Нет packet-логирования.** В прошивке нет `xt_LOG`, так что event log пишет *изменения*
  членства (ban / release / fold), а не отдельные пакеты. Графики dashboard выводят объём
  drop из кумулятивного счётчика.
- **Enforcement по IP.** Упорный атакующий, ротирующий много адресов, может продолжать
  зондировать; вы можете ответить баном всей его subnet одной записью, но это не замена
  аутентификации и патчингу сервисов, которые вы выставляете.
- **Флаг страны/владельца может отставать на пару минут.** Большие geo/owner-базы
  резолвятся с периодичностью 5 минут (`GEO_EVERY`, по умолчанию 5); *бан* немедленный,
  только косметический флаг/owner-label появляется чуть позже.

## Security-заметки

- **У UI нет аутентификации.** Он слушает только на LAN-IP, порт 5050, и предназначен для
  доверенной домашней LAN, как админка самого роутера. **Не пробрасывайте его порт в
  интернет.** API принимает запросы из LAN без CSRF-токена, так что вредоносный сайт,
  открытый в LAN, в принципе может им управлять (например, `<img>`, стреляющий GET). Это
  свойство модели no-auth LAN-appliance, а не дефект; митигация — доверенная LAN и не
  выставлять порт наружу. Все действия обратимы из UI или shell.
- **Весь ввод, что становится ban- или whitelist-записью, проверяется** как IP или CIDR
  (octets 0–255, mask 8–32 — блок `/0`–`/7` никогда не принимается) прежде чем коснуться
  `ipset`. Имена и параметры API-эндпоинтов
  ограничены известным набором; инъекция через поле адреса отклоняется.
- **Free-text whitelist-notes очищаются** — кавычки, backslash, структурные и
  control-символы вырезаются — так что note (заданная в UI, импортированная из backup или
  вписанная руками в файл) никогда не сломает JSON-payload или import-парсер.
- **Никакие данные не покидают роутер.** Все threat/geo/owner-данные скачиваются и
  резолвятся локально; UI не контактирует ни с чем, кроме собственного CGI роутера.
  Единственные исходящие соединения Tacet — скачивания баз (на jsDelivr / источники feeds)
  и GitHub-release-check для обновлений.
